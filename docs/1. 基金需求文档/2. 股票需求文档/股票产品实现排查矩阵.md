# 股票产品实现排查矩阵

排查日期: 2026-06-19

## 1. 结论

当前股票端不是单点缺陷，而是需求、前端、后端、数据闭环没有统一收敛。

- 需求目录按 67 个菜单展开，覆盖数据中心、市场行情、量化指标、回测交易、AI 研究和系统平台。
- 前端 `/stock` 实际承载 10 个页面: 工作台、自选股、行情、K线分析、选股、回测、模拟交易、告警、数据中心、设置。
- 后端 `tradehub-stock` 已有自选、行情快照、K线、ETF 风控、指标/形态、选股、回测、策略模板、模拟交易、告警、任务等基础接口。
- 产品化问题主要集中在: 页面数量远少于需求菜单、部分页面使用静态演示数据、组合选股只执行单条件、数据中心接口与页面未打通、AI 研究和市场资讯类能力基本未落地。

## 4. 已确认的工程问题

### P0: 回测摘要模型漂移

现象:

- `model.BacktestSummary` 使用 `Metrics.TotalTrades`。
- 测试仍断言 `BacktestSummary.TotalTrades`。
- 数据库摘要表仍是扁平字段 `total_trades`, `win_rate`, `total_return_pct` 等。

处理:

- 保留 API 模型的 `metrics` 结构。
- Repository 将数据库扁平字段映射进 `summary.metrics`。
- `best_return_pct`, `worst_return_pct`, `benchmark_*`, `return_distribution` 等非核心指标暂放入 `meta_json`。

### P0: 数据中心页面是假数据

位置:

- `tradehub-frontend/src/stock/pages/DataCenterPage.jsx`

问题:

- `DATA_SOURCES` 为静态常量。
- `SCHEDULED_TASKS` 为静态常量。
- 包含 `每日净值同步` 这种基金语义。
- 页面没有读取 `/api/stock/v1/datacenter/health`, `/api/stock/v1/datacenter/tasks`, `/api/stock/v1/tasks` 的真实状态。

整改:

- 删除静态任务和静态统计。
- 增加 loading、error、empty 状态。
- 接后端真实数据中心健康、采集任务、通用任务列表。

### P0: 选股页面没有执行组合条件

位置:

- `tradehub-frontend/src/stock/pages/ScreenerPage.jsx`

问题:

- 指标和形态选项写死在前端。
- 执行选股时只使用 `conditions[0]`。
- 价格、资金、基本面条件在页面上出现，但后端当前没有同等查询能力。
- 保存模板的条件结构和后端任务条件树没有明确对齐。

整改:

- 指标定义和形态定义从后端加载。
- 单条件查询只保留为快速筛选。
- 多条件改为创建 `screening` 任务，结果从 `/screener/results` 查询。
- 前端条件树统一为 `indicator_conditions`, `pattern_conditions`, `logic`。

### P1: DataCenterHandler 路由重复

位置:

- `tradehub-stock/internal/interfaces/httpapi/datacenter_handler.go`
- `tradehub-stock/internal/interfaces/httpapi/datacenter_handler/handler.go`

问题:

- 同名 handler 存在两套文件组织。
- 路由同时出现 `/datacenter/query/daily` 和 `/datacenter/daily-spot` 两种风格。
- 容易造成文档、OpenAPI、前端调用漂移。

整改:

- 保留一个 handler 入口。
- 统一路由命名。
- OpenAPI、前端 client、README 同步。

### P1: 旧股票页面和新股票页面并存

位置:

- `tradehub-frontend/src/pages/Stock*.jsx`
- `tradehub-frontend/src/stock/pages/*.jsx`

问题:

- `/dashboard/stock/*` 和 `/stock/*` 两套股票页面并存。
- 新布局 `StockLayout` 承载专业股票路由，旧页面仍留在主 dashboard 路由。
- 容易出现同一功能两个入口、两个 API client、两个状态口径。

整改:

- 明确 `/stock/*` 为股票端唯一主入口。
- 旧 `/dashboard/stock/*` 只保留跳转或逐步删除。

### P0: screening 任务流与 TDengine REST 阻塞

位置:

- `tradehub-stock/worker/main.go`
- `tradehub-stock/worker/marketapi.go`
- `tradehub-stock/api/main.go`
- `tradehub-stock/api/marketapi.go`
- `tradehub-frontend/vite.config.js`
- `dev.sh`
- `docker-compose.local.yml`

原现象:

- 前端已能成功创建 `screening` 任务，`POST /api/stock/v1/tasks` 返回 `pending`。
- worker 本地未运行时，任务一直停留在 `pending`。
- 已补 `./dev.sh worker` 启动入口，但 worker 启动时在 TDengine 初始化阶段失败。
- 宿主机 `127.0.0.1:16042` 端口可连，但 `POST /rest/sql` 超时；容器日志显示 `taosadapter` 在监听 `6041`，同时 `GET /rest/sql` 返回 404，`POST /rest/sql` 未带认证会返回 `no auth info input`。
- 容器内 `taos` 原生命令可进入，说明 TDengine 核心服务不等于完全不可用，阻塞点是当前 REST 访问链路。

已处理:

- 2026-06-19 非破坏性重启 `tradehub-tdengine` 后，`taosd`、`taosadapter`、`taoskeeper` 均恢复监听，TDengine 状态变为 `healthy`。
- 修复 TDengine `indicator_values` 超级表字段名，避开当前 TDengine 版本对 `value` 列名的语法错误，使用 `indicator_value` 存储并在 API 返回中补回 `value` 兼容前端。
- 修复 `stock_tasks` 状态更新 SQL 的 `$1` 类型推断问题，避免任务失败后无限停留在 `pending`。
- `./dev.sh worker` 默认补齐 `MARKET_API_URL=http://127.0.0.1:17080`，A 股日 K 采集优先走 market-api，不再因为本机缺 `python3.11/akshare` 直接失败。
- market-api 日 K 路径增加腾讯、东财、搜狐 fallback；当前腾讯日 K 上游可出现 `unexpected end of JSON input`，实际已通过东财 `em_day` 成功落库。
- Vite 股票 API 代理改为 `VITE_STOCK_API_TARGET` 可配置，当前默认指向源码 API `http://127.0.0.1:7003`，避免继续命中旧 `7002` 容器二进制。

验证:

- `POST http://127.0.0.1:16042/rest/sql` 可返回 `stock_etf_ts`。
- 采集任务 `task_1781710383461002250` 状态为 `completed`，消息为 `inserted 240 daily bars`。
- TDengine `stock_etf_ts.bars_000001_1d` 当前有 240 行，最新数据来源 `em_day`。
- 指标任务 `stock_task_1781883348037394000` 状态为 `succeeded`，结果写入 `tdengine:stock_etf_ts.ind_000001_1d_ma`。
- 选股任务 `stock_task_1781883419331816000` 状态为 `succeeded`，`stock_screening_results` 返回 1 条命中结果。

剩余注意:

- 旧 Docker 业务容器 `tradehub-stock-api` 仍占用 `7002`，它不是当前源码构建，仍可能返回旧 SQL 错误。当前前端通过 Vite 代理使用 `7003` 源码 API。
- 若后续切回 `7002`，需要重建业务容器或改为 host 方式启动 `./dev.sh stock`。

## 5. 暂不进入第一阶段的能力

以下需求文档先保留，不作为当前交付口径:

- AI 分析报告、AI 推荐记录、AI 智能体、MCP 服务、提示词广场、问答广场。
- 自动交易真实下单。
- 问财自然语言选股。
- F10 基本面全量选股。
- 研报、公告、行业研究、财经日历、滚动快讯。
- CYQ 筹码分布完整复刻。

这些能力需要单独的数据源、授权、风控和异步任务设计，不能靠前端页面占位解决。
