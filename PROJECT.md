# TradeHub 项目说明

## 项目定位

TradeHub 是一个个人量化投资与投研一体化平台，目标是把基金、股票、行情、量化分析、策略回测、模拟交易、AI 投研和自动化任务放到同一个产品体系里管理。

它不是单纯的基金记账工具，也不是只看股票行情的看盘页面。TradeHub 更接近一个本地/私有化部署的投资研究工作台：持续同步数据、沉淀结构化信息、跟踪持仓和自选、分析排行和板块变化，并为后续 AI Agent 投研能力提供数据底座。

## 项目特点

### 1. 多源数据拉取

TradeHub 的数据设计不是绑定单一接口，而是面向多源聚合和容错。

基金侧会组合使用东方财富、腾讯基金、养基宝、小倍养基等来源。不同来源承担不同职责：有的适合拉基金列表和历史净值，有的适合拉基金资料、基金经理、基金公司、规模、行业配置和持仓，有的适合补充用户持仓或社区信息。单个数据源失败时，任务应记录失败原因并继续处理其他来源，避免整批同步中断。

股票侧通过 `tradehub-market-api` 聚合腾讯、东方财富、搜狐、同花顺、雪球等行情源，为股票服务提供统一的行情、K 线、快照、板块和市场数据入口。业务模块不直接散落调用第三方接口，后续可以统一做缓存、限流、降级和源健康检查。

### 2. 数据不是只展示，而是结构化入库

项目强调把可复用的数据沉淀到系统表里，而不是每次页面打开都临时请求外部 API。

典型入库内容包括：

- 基金基础信息、基金公司、基金经理、基金规模。
- 基金历史净值、每日事实表、盘中估值、实际收益。
- 基金排行，支持日、周、月、今年、近一年等周期。
- 基金持仓、成分股、行业配置、板块归属。
- 市场板块涨跌快照，支持盘中和盘后分析。
- 股票行情、K 线、指标、告警、模拟交易和策略结果。

这样做的价值是数据可追踪、可复盘、可筛选、可用于后续 AI 分析和自动任务。

### 3. 信源和日期可追溯

投研数据必须知道“从哪里来”和“是哪一天的数据”。TradeHub 在基金排行、净值、社区信息、板块快照、工具调用和 AI 输出中都强调信源标记。

系统后续会持续强化：

- 每条关键数据保留 `source`、`trade_date`、`snapshot_time`。
- 页面展示数据来源和更新时间。
- AI/Agent 输出必须引用数据日期和来源。
- 第三方源失败时记录错误，不把旧数据伪装成新数据。

### 4. 自动化任务驱动

TradeHub 的核心数据依赖定时任务持续刷新，而不是靠人工点按钮。

典型任务包括：

- 盘中同步行情、估值和板块快照。
- 下午和收盘后同步确认净值、排行、板块和收益。
- 晚间补全基金资料、持仓、规模和历史净值。
- 股票行情、K 线、指标、策略和告警任务周期执行。
- 任务失败时记录错误，支持后续重试和排查。

### 5. 基金和股票统一产品体验

项目同时覆盖基金和股票，但不是两套割裂系统。前端使用统一产品壳、统一登录、统一设置、统一自选/关注思路，后端共享基础设施、任务体系、AI 配置和审计能力。

基金和股票可以有各自业务模型，但交互方式、菜单组织、颜色规则、数据筛选、加入自选、详情跳转和顶部操作会逐步统一。

### 6. 为 AI Agent 准备的数据底座

TradeHub 的长期特点是让 AI 不只读用户输入，而是能读取本地结构化投研数据。

后续 Agent 能力会基于：

- 已入库的基金和股票数据。
- 明确的数据来源和日期。
- 可调用的内部工具和行情 API。
- 可版本化的提示词、规则和技能。
- 每个子 Agent 自己的公众号、Markdown、文章和 Git 知识库。

这使 Agent 可以做更可追溯的基金分析、板块轮动、公告解读、策略复盘和自动报告。

## 主要能力

### 1. 基金投研

基金模块用于管理基金基础信息、净值、估值、排行、持仓、基金经理、基金公司、基金规模和板块归因。

当前重点能力包括：

- 基金列表、详情、自选、持仓、账户管理。
- 基金净值历史和每日事实表入库。
- 基金排行入库，支持日、周、月、今年、近一年等周期。
- 基金公司、基金经理、基金规模、行业配置、成分股持仓等结构化信息。
- 基金所属板块、市场板块涨跌和板块轮动分析。
- 基金社区信息动态查看，保留信源标记。
- 盘中估值、盘后实际净值、估值准确率和收益信息。

### 2. 股票投研

股票模块用于承载股票自选、实时行情、K 线、盘口、订单、告警、数据入库、策略和回测等能力。

当前规划和部分实现能力包括：

- 股票自选分组和关注列表。
- 股票实时行情、历史行情、K 线图和盘口信息。
- 股票账户、订单、模拟交易和风险视图。
- 股票数据中心、数据入库、指标数据和行情采集任务。
- 股票告警、策略运行、回测和推荐记录。
- 行业、板块、市场行情和全球市场入口。

### 3. 行情源聚合

`tradehub-market-api` 是行情网关，负责统一接入第三方行情源，避免前端和业务模块直接散落调用外部接口。

当前定位：

- 聚合腾讯、东方财富、搜狐、同花顺、雪球等数据源。
- 为股票服务提供行情、K 线、快照、板块等数据。
- 做源适配、缓存、超时控制、健康检查和后续数据质量治理。

### 4. 数据入库与自动化任务

TradeHub 不只做页面展示，而是强调数据落库和可追溯。

核心数据层：

- PostgreSQL：基金、股票、账户、自选、持仓、任务、排行、审计等业务数据。
- TDengine：股票 K 线、分钟线、资金流等时序数据。
- Redis：缓存、锁、Celery 队列、短期任务状态。

自动任务方向：

- 基金列表、净值、排行、资料、持仓、板块快照定时同步。
- 股票行情、K 线、指标和策略任务定时执行。
- 盘中快照、盘后收盘数据、夜间补全数据分时段调度。
- 任务执行结果、失败原因、数据来源和更新时间可追踪。

### 5. AI 投研与 Agent 能力

项目正在扩展 AI 研究中心和 Agent 能力。目标不是简单接一个聊天框，而是让 AI 可以基于本地结构化数据、提示词、工具和规则执行投研任务。

规划能力包括：

- AI 模型配置，支持 OpenAI 协议兼容接口。
- 提示词模板、报告生成、问答和推荐记录。
- Agent Skill：可复用投研技能，例如基金分析、公告解读、板块轮动。
- Agent Rules：行为规则、工具权限、输出约束、风控规则。
- Agent API：把 Agent 暴露给页面、定时任务和外部调用。
- 每个子 Agent 拥有自己的知识源，例如公众号、Markdown、文章、Git 仓库。
- 从知识中抽取核心提示词、方法论、工具需求和输出模板。
- Agent Package 标准化，后续可提交 Git 或社区仓库。

## 系统模块

| 模块 | 路径 | 主要职责 |
| --- | --- | --- |
| 前端 | `tradehub-frontend/` | React + Vite 统一前端，基金、股票、设置、AI 等页面 |
| 基金后端 | `tradehub-fund/` | Django + DRF，基金、账户、持仓、通知、AI 配置、Celery |
| 股票后端 | `tradehub-stock/` | Go API + Worker，股票业务、采集、告警、模拟交易、策略 |
| 行情网关 | `tradehub-market-api/` | Go 行情源聚合，统一第三方行情接入 |
| 基础设施 | `docker-compose*.yml` | PostgreSQL、Redis、TDengine 和一体化编排 |
| 文档 | `docs/` | 需求、架构、API、Agent 设计和产品排查文档 |

## 产品边界

TradeHub 当前坚持“大服务、强模块边界、关键能力进程级解耦”的路线。

它会保留前端、基金后端、股票后端、行情网关等独立进程，但第一阶段不引入完整微服务治理，不做服务注册中心、服务网格、复杂分布式事务或过早 Kubernetes 化。

模块边界：

- 前端负责交互、图表、页面状态，不直接调用第三方行情源。
- 基金后端负责基金、账户、持仓、通知、基金 AI 能力。
- 股票后端负责股票、自选、行情聚合、指标、回测、模拟交易。
- 行情网关负责第三方行情源适配，不承载用户业务。
- 数据库按业务数据、时序数据、缓存和任务状态分层。

## 当前适用场景

TradeHub 适合这些使用方式：

- 个人搭建一个私有化投资数据平台。
- 跟踪基金持仓、净值、排行、规模、行业和板块变化。
- 跟踪股票自选、行情、K 线、告警和模拟交易。
- 做基金和股票数据的每日自动同步。
- 对基金排行、板块轮动和股票策略做持续观察。
- 为 AI 投研、Agent 分析和自动报告建立本地数据基础。

## 非目标

当前阶段不以这些目标为优先：

- 不做真实自动交易下单。
- 不做完整证券交易终端替代品。
- 不承诺第三方行情源 100% 稳定。
- 不把所有 AI 输出视为投资建议。
- 不在第一阶段做完整微服务化和复杂云原生治理。

## 当前状态

项目处于快速迭代阶段，基金和股票能力都在补齐数据结构、页面交互、自动同步和部署稳定性。

当前最重要的建设方向：

- 基金数据稳定落库，包括排行、净值、规模、持仓、板块和日事实。
- 股票菜单和基金菜单交互统一。
- 股票行情、数据中心、K 线、告警、模拟交易和策略能力补齐。
- 定时任务稳定化，减少手工刷数据。
- 新机器部署和服务运维文档完善。
- Agent 能力中心设计和后续落地。

## 一句话总结

TradeHub 是一个面向个人投资研究的私有化数据与 AI 工作台：用统一前端管理基金和股票，用结构化数据支撑排行、板块、持仓、行情和策略分析，并逐步接入可配置、可审计、可社区化的 AI Agent 投研能力。
