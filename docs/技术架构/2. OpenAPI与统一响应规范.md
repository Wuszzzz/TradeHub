# OpenAPI 与统一响应规范

## 1. 目标

统一 TradeHub 后续新增 API 的命名、版本、响应、错误码、分页、批量任务格式。第一阶段先定规范，旧接口保留兼容，新功能按新规范实现。

## 2. API 分域

| 域 | 路径前缀 | 说明 |
| --- | --- | --- |
| 基金 | `/api/fund/v1` | 基金列表、详情、净值、估值、自选、持仓、账户 |
| 股票 | `/api/stock/v1` | 股票自选、行情聚合、告警、模拟交易、个股信息 |
| 行情 | `/api/market/v1` | 行情源能力、快照、K 线、资金流、盘口、源健康 |
| 量化 | `/api/quant/v1` | 指标、K 线形态、形态选股、指标选股 |
| 回测 | `/api/backtest/v1` | 策略回测、结果、指标、交易明细 |
| AI 研究 | `/api/ai/v1` | AI 配置、Prompt、研报、对话、Agent 任务 |
| 系统 | `/api/system/v1` | 健康检查、任务、审计、设置、字典 |

## 4. 统一响应

### 4.1 成功响应

```json
{
  "success": true,
  "code": "OK",
  "message": "ok",
  "data": {
    "symbol": "600519",
    "name": "贵州茅台"
  },
  "meta": {
    "request_id": "req_20260614_120000_abcd",
    "timestamp": "2026-06-14T12:00:00+08:00",
    "version": "v1"
  }
}
```

### 4.2 失败响应

```json
{
  "success": false,
  "code": "STOCK_SYMBOL_NOT_FOUND",
  "message": "标的不存在",
  "data": null,
  "meta": {
    "request_id": "req_20260614_120000_abcd",
    "timestamp": "2026-06-14T12:00:00+08:00",
    "version": "v1"
  },
  "error": {
    "field": "symbol",
    "detail": "6000000 is not a valid symbol"
  }
}
```

### 4.3 列表分页响应

```json
{
  "success": true,
  "code": "OK",
  "message": "ok",
  "data": {
    "items": [],
    "pagination": {
      "page": 1,
      "page_size": 20,
      "total": 128,
      "has_next": true
    }
  },
  "meta": {
    "request_id": "req_20260614_120000_abcd",
    "timestamp": "2026-06-14T12:00:00+08:00"
  }
}
```

### 4.4 游标分页响应

行情、逐笔、日志、事件流建议使用游标分页：

```json
{
  "success": true,
  "code": "OK",
  "message": "ok",
  "data": {
    "items": [],
    "cursor": {
      "next": "2026-06-14T10:30:00+08:00:600519",
      "has_next": true
    }
  },
  "meta": {
    "request_id": "req_20260614_120000_abcd",
    "timestamp": "2026-06-14T12:00:00+08:00"
  }
}
```

## 5. 批量任务响应

耗时任务不能同步返回大结果，统一返回任务对象。

```json
{
  "success": true,
  "code": "TASK_CREATED",
  "message": "任务已创建",
  "data": {
    "task_id": "task_20260614_kpattern_0001",
    "task_type": "kline_pattern_scan",
    "status": "pending",
    "progress": 0,
    "created_at": "2026-06-14T12:00:00+08:00"
  },
  "meta": {
    "request_id": "req_20260614_120000_abcd",
    "timestamp": "2026-06-14T12:00:00+08:00"
  }
}
```

任务状态：

| 状态 | 说明 |
| --- | --- |
| `pending` | 已创建，等待调度 |
| `running` | 执行中 |
| `succeeded` | 成功 |
| `failed` | 失败 |
| `retrying` | 重试中 |
| `cancelled` | 已取消 |

## 6. 错误码

### 6.1 通用错误码

| code | HTTP | 说明 |
| --- | --- | --- |
| `OK` | 200 | 成功 |
| `BAD_REQUEST` | 400 | 请求参数错误 |
| `UNAUTHORIZED` | 401 | 未登录或 token 失效 |
| `FORBIDDEN` | 403 | 权限不足 |
| `NOT_FOUND` | 404 | 资源不存在 |
| `CONFLICT` | 409 | 资源冲突 |
| `VALIDATION_ERROR` | 422 | 字段校验失败 |
| `RATE_LIMITED` | 429 | 请求过快 |
| `INTERNAL_ERROR` | 500 | 服务内部错误 |
| `UPSTREAM_ERROR` | 502 | 上游数据源失败 |
| `SERVICE_UNAVAILABLE` | 503 | 服务不可用 |

### 6.2 股票错误码

| code | 说明 |
| --- | --- |
| `STOCK_SYMBOL_INVALID` | 股票代码格式错误 |
| `STOCK_SYMBOL_NOT_FOUND` | 标的不存在 |
| `STOCK_MARKET_UNSUPPORTED` | 市场暂不支持 |
| `STOCK_WATCHLIST_DUPLICATED` | 自选重复 |
| `STOCK_ALERT_RULE_INVALID` | 告警规则不合法 |
| `STOCK_PAPER_CASH_NOT_ENOUGH` | 模拟账户现金不足 |
| `STOCK_PAPER_POSITION_NOT_ENOUGH` | 模拟账户持仓不足 |

### 6.3 行情错误码

| code | 说明 |
| --- | --- |
| `MARKET_SOURCE_TIMEOUT` | 行情源超时 |
| `MARKET_SOURCE_RATE_LIMITED` | 行情源限流 |
| `MARKET_SOURCE_UNAUTHORIZED` | 需要 Cookie 或授权 |
| `MARKET_DATA_EMPTY` | 上游返回空数据 |
| `MARKET_DATA_DEGRADED` | 已降级返回 |

### 6.4 量化错误码

| code | 说明 |
| --- | --- |
| `QUANT_INDICATOR_UNSUPPORTED` | 指标暂不支持 |
| `QUANT_PATTERN_UNSUPPORTED` | K 线形态暂不支持 |
| `QUANT_PARAMETER_INVALID` | 指标参数错误 |
| `QUANT_DATA_NOT_READY` | 计算所需行情数据不足 |

## 7. 请求规范

### 7.1 request_id

服务端若请求头没有 `X-Request-ID`，自动生成：

```text
req_{yyyyMMdd_HHmmss}_{random}
```

响应 `meta.request_id` 必须带回。

### 7.2 时间

所有 API 返回时间使用 ISO-8601：

```text
2026-06-14T12:00:00+08:00
```

行情源原始时间可以保留在 `source_time` 字段，但业务时间统一转换。

### 7.3 金额和价格

| 字段类型 | 规则 |
| --- | --- |
| 价格 | decimal string 或 number，前端展示按精度格式化 |
| 成交量 | 明确单位，字段名带 `share`、`hand` 或 `unit` |
| 成交额 | 字段名带 `yuan` |
| 比率 | 百分比字段名带 `percent`，值为百分数，不是小数 |

## 8. 兼容策略

| 当前接口 | 新接口方向 | 处理 |
| --- | --- | --- |
| `/api/v1/instruments/*` | `/api/stock/v1/instruments/*` | 保留旧接口，新增新路径 |
| `/api/v1/market/*` | `/api/stock/v1/market/*` | 保留旧接口，内部调用同一 application |
| `/api/v1/alerts/*` | `/api/stock/v1/alerts/*` | 保留旧接口 |
| `/api/v1/paper/*` | `/api/stock/v1/paper/*` | 保留旧接口 |
| `market-api /api/v1/tencent/*` | `/api/market/v1/sources/tencent/*` | 第一阶段暂不强迁移 |

## 9. OpenAPI 最小样例

```yaml
openapi: 3.1.0
info:
  title: TradeHub API
  version: 1.0.0
paths:
  /api/stock/v1/instruments/search:
    get:
      summary: 搜索股票标的
      parameters:
        - name: keyword
          in: query
          required: true
          schema:
            type: string
      responses:
        "200":
          description: OK
```
