# 数据库分层设计

## 1. 目标

TradeHub 后续要承载基金、股票、行情、指标、形态、选股、回测、AI 报告等能力。第一阶段先明确数据分层，避免所有数据都塞进 PostgreSQL 或所有行情都临时算。

## 2. 总体分层

| 层 | 存储 | 职责 |
| --- | --- | --- |
| 业务配置层 | PostgreSQL | 用户、账户、自选、策略、告警、任务、AI 配置 |
| 时序行情层 | TDengine | K 线、分钟线、逐笔、资金流、指标序列 |
| 缓存锁层 | Redis | 实时快照缓存、任务锁、限流、短状态 |
| 源数据层 | PostgreSQL raw 表或文件 | 第三方原始响应、源请求审计 |
| 产物层 | 本地文件或对象存储 | 回测报告、AI 研报、CSV/Excel 导出 |

### 4.2 统一任务表

`stock_tasks`

| 字段 | 说明 |
| --- | --- |
| `task_id` | 任务 ID |
| `task_type` | `ingest` / `indicator` / `pattern` / `screen` / `backtest` / `ai_report` |
| `status` | `pending` / `running` / `succeeded` / `failed` / `retrying` / `cancelled` |
| `params` | JSON 参数 |
| `result_ref` | 结果引用 |
| `progress` | 进度 |
| `attempts` | 重试次数 |
| `last_error` | 最近错误 |
| `created_by` | 创建人 |
| `created_at` | 创建时间 |
| `started_at` | 开始时间 |
| `finished_at` | 结束时间 |

当前第一期已经落地 `stock_tasks` 表，支持以下 `task_type`：

| task_type | 说明 |
| --- | --- |
| `data_ingest` | 行情或基础数据采集任务 |
| `indicator_compute` | 技术指标计算任务 |
| `pattern_scan` | K 线形态扫描任务 |
| `screening` | 条件选股任务 |
| `backtest` | 策略回测任务 |
| `ai_report` | AI 研报生成任务 |

任务状态：

| status | 说明 |
| --- | --- |
| `pending` | 已创建，等待执行 |
| `running` | 执行中 |
| `succeeded` | 成功 |
| `failed` | 失败 |
| `retrying` | 等待重试 |
| `cancelled` | 已取消 |

### 4.3 指标定义

`stock_indicator_definitions`

| 字段 | 说明 |
| --- | --- |
| `indicator_code` | 指标编码，如 `MACD`、`RSI`、`BOLL` |
| `name` | 中文名称 |
| `category` | 趋势、震荡、成交量、波动率 |
| `description` | 指标说明 |
| `params_schema` | 参数 schema |
| `output_fields` | 输出字段，如 `dif/dea/macd`、`kdjk/kdjd/kdjj` |
| `enabled` | 是否启用 |

当前已落地 PostgreSQL 表和启动种子：`MA`、`EMA`、`MACD`、`KDJ`、`BOLL`、`RSI`、`TRIX`、`VR`、`WR`。后续新增指标只需要追加定义，不应该改页面硬编码。

### 4.4 K 线形态定义

`stock_pattern_definitions`

| 字段 | 说明 |
| --- | --- |
| `pattern_code` | 形态编码 |
| `name` | 中文名 |
| `category` | 形态分类，当前为 `candlestick` |
| `talib_function` | TA-Lib 计算函数，如 `CDLENGULFING` |
| `direction` | `bullish` / `bearish` / `both` / `neutral` |
| `description` | 说明 |
| `params_schema` | 可调参数 |
| `enabled` | 是否启用 |

61 种 K 线形态已作为启动种子进入 PostgreSQL 定义表，不写死在页面筛选项里。形态计算和命中结果仍走 `pattern_scan` 任务与 TDengine `pattern_hits`。

### 4.5 组合选股模板

`stock_screening_templates`

| 字段 | 说明 |
| --- | --- |
| `template_id` | 模板 ID |
| `name` | 模板名称 |
| `description` | 模板说明 |
| `conditions` | JSON 条件树，保存指标条件、形态条件、逻辑关系 |
| `enabled` | 是否启用 |
| `created_by` | 创建人 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

`stock_screening_results`

| 字段 | 说明 |
| --- | --- |
| `result_id` | 结果 ID |
| `task_id` | 关联 `stock_tasks.task_id` |
| `template_id` | 来源模板 ID，内联条件任务可为空 |
| `symbol` | 命中标的 |
| `score` | 命中得分，当前为 `命中条件数 / 总条件数` |
| `matched_conditions` | 命中的指标或形态条件快照 |
| `snapshot` | 用于列表展示的最新条件快照 |
| `created_at` | 创建时间 |

第一期已把模板作为 PostgreSQL 业务配置，支持 HTTP CRUD。批量选股通过 `stock_tasks.task_type=screening` 异步执行，结果写入 `stock_screening_results`，避免 HTTP 请求承担长耗时计算。

### 4.6 策略模板

`stock_strategy_templates`

| 字段 | 说明 |
| --- | --- |
| `strategy_id` | 策略模板 ID |
| `name` | 策略名称 |
| `description` | 策略说明 |
| `screening_template_id` | 可选的选股模板引用 |
| `conditions` | 内联选股条件 |
| `backtest_params` | 回测参数，如周期、持有根数、基准 |
| `risk_params` | 风控参数，如手续费、滑点、止损、止盈 |
| `enabled` | 是否启用 |
| `created_by` | 创建人 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

`backtest` 任务可通过 `strategy_id` 引用策略模板。任务参数同名字段优先级高于模板字段，便于临时覆盖。

任务创建时会把 `strategy_id` 冻结为 `strategy_snapshot` 写入 `stock_tasks.params`，历史任务不依赖后续模板修改。

`stock_strategy_runs`

| 字段 | 说明 |
| --- | --- |
| `run_id` | 运行记录 ID |
| `task_id` | 关联 `stock_tasks.task_id` |
| `task_type` | `screening` / `backtest` |
| `strategy_id` | 策略模板 ID |
| `snapshot_id` | 冻结快照 ID |
| `strategy_name` | 运行时策略名称 |
| `status` | 当前任务状态 |
| `result_ref` | 结果引用 |
| `summary_ref` | 汇总结果引用 |
| `created_by` | 创建人 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

`stock_strategy_runs` 用来串联模板、快照、任务和结果，避免历史回测只能从 `stock_tasks.params` 里反查。

### 4.7 回测结果

`stock_backtest_results`

| 字段 | 说明 |
| --- | --- |
| `result_id` | 结果 ID |
| `task_id` | 关联 `stock_tasks.task_id` |
| `symbol` | 回测标的 |
| `period` | K 线周期 |
| `entry_time` | 入场时间 |
| `exit_time` | 出场时间 |
| `entry_price` | 入场价格 |
| `exit_price` | 出场价格 |
| `return_pct` | 扣除滑点和双边手续费后的净收益率 |
| `benchmark_symbol` | 基准标的 |
| `benchmark_return_pct` | 基准同期收益率 |
| `excess_return_pct` | 相对基准超额收益率 |
| `meta` | 回测参数、来源等上下文 |
| `created_at` | 创建时间 |

`stock_backtest_summaries`

| 字段 | 说明 |
| --- | --- |
| `summary_id` | 汇总 ID |
| `task_id` | 关联 `stock_tasks.task_id` |
| `total_trades` | 交易 / 标的数量 |
| `win_rate` | 胜率 |
| `avg_return_pct` | 平均净收益率 |
| `total_return_pct` | 复合总收益率 |
| `max_drawdown_pct` | 基于任务结果序列估算的最大回撤 |
| `best_return_pct` | 最佳单标的收益 |
| `worst_return_pct` | 最差单标的收益 |
| `benchmark_symbol` | 基准标的 |
| `benchmark_return_pct` | 基准收益率 |
| `avg_excess_return_pct` | 平均超额收益率 |
| `return_distribution` | 收益分布桶 |
| `meta` | 回测参数、来源等上下文 |
| `created_at` | 创建时间 |

第一期回测先作为异步任务产物表落地，支持 `symbols`、`screening_task_id`、`template_id/conditions` 三种标的来源。当前计算口径是最近 `hold_bars` 根 K 线持有回测，已支持 `fee_rate`、`slippage_rate`、`stop_loss`、`take_profit`、`benchmark_symbol`，并输出组合级胜率、收益、最大回撤、收益分布和超额收益；后续再扩展到完整交易撮合、调仓、资金曲线和组合净值。

## 5. TDengine 时序模型

### 5.1 K 线超级表

`market_bars`

| 字段 | 说明 |
| --- | --- |
| `ts` | 时间 |
| `open` | 开盘价 |
| `high` | 最高价 |
| `low` | 最低价 |
| `close` | 收盘价 |
| `volume` | 成交量 |
| `amount` | 成交额 |
| `turnover_rate` | 换手率 |
| `source` | 数据源 |

标签：

| tag | 说明 |
| --- | --- |
| `symbol` | 标的代码 |
| `period` | `1m` / `5m` / `15m` / `30m` / `60m` / `1d` |
| `adjust` | `none` / `qfq` / `hfq` |
| `market` | 市场 |

### 5.2 指标序列超级表

`indicator_values`

| 字段 | 说明 |
| --- | --- |
| `ts` | 时间 |
| `value` | 主值 |
| `values_json` | 多输出指标 JSON，如 MACD dif/dea/bar |

标签：

| tag | 说明 |
| --- | --- |
| `symbol` | 标的代码 |
| `period` | 指标对应 K 线周期 |
| `indicator_code` | 指标编码，如 `MACD`、`KDJ`、`BOLL` |

当前 `indicator_compute` 已由 worker 执行，读取 `market_bars` 子表，计算 `MA/EMA/MACD/KDJ/BOLL/RSI/TRIX/VR/WR`，写入 `indicator_values` 子表 `ind_<symbol>_<period>_<indicator>`。

标签：

| tag | 说明 |
| --- | --- |
| `symbol` | 标的 |
| `period` | 周期 |
| `indicator_code` | 指标 |
| `params_hash` | 参数哈希 |

### 5.3 形态命中超级表

`pattern_hits`

| 字段 | 说明 |
| --- | --- |
| `ts` | 形态命中 K 线时间 |
| `pattern_value` | 形态值，`-100` / `100`，兼容 TA-Lib 语义 |
| `direction` | `bullish` / `bearish` / `neutral` |
| `extra_json` | 形态上下文 |
| `algorithm_version` | 算法版本，当前为 `go-candlestick-v1` |

标签：

| tag | 说明 |
| --- | --- |
| `symbol` | 标的 |
| `period` | 周期 |
| `pattern_code` | 形态编码 |

当前 `pattern_scan` 已由 worker 执行，读取 `market_bars` 子表，扫描 61 种形态编码，命中写入 `pattern_hits` 子表 `pattern_<symbol>_<period>_<pattern>`。第一版使用 Go 本地规则引擎，后续可替换为 TA-Lib 精确引擎，但任务、表和 API 契约不变。

## 6. Redis 使用规范

| key | TTL | 说明 |
| --- | --- | --- |
| `stock:snapshot:{symbol}` | 5-10s | 实时快照 |
| `stock:rank:{rank_type}:{date}` | 30-120s | 排行缓存 |
| `stock:task:lock:{task_id}` | 任务执行期 | 防重复执行 |
| `market:source:health:{source}` | 30s | 行情源健康 |
| `rate:{user_id}:{api}` | 1-60s | API 限流 |

Redis 不保存长期业务事实，只保存可丢弃、可重建的数据。

## 7. 数据生命周期

| 数据 | 保留策略 |
| --- | --- |
| 实时快照缓存 | 秒级 TTL |
| 1m/5m 分钟线 | 默认保留 60-180 天 |
| 日线 | 长期保留 |
| 指标序列 | 按自选/策略范围保留 |
| 形态命中 | 可按周期归档 |
| 回测结果 | 保留结果索引，产物文件可清理 |
| 源请求审计 | 保留 7-30 天 |
| API 审计 | 保留 30-90 天 |

## 8. 第一阶段落库边界

第一阶段只需要先建或规划：

- `stock_tasks`
- `stock_task_logs`
- `stock_instruments`
- `stock_indicator_definitions`
- `stock_pattern_definitions`
- `stock_data_source_audits`
- TDengine `market_bars`
- TDengine `indicator_values`
- TDengine `pattern_hits`

不要第一阶段就全市场大规模入库。先按自选、策略、回测任务需要进行按需采集和补齐。
