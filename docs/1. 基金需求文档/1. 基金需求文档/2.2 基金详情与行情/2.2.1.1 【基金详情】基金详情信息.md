## 📌 业务目标

- 基金代码

- 基金类型
- 基金规模

- 基金披露最新涨跌幅

- 基金的业绩表现（历史净值、收益率等）

- 基金成立时间
- 基金k线
- 基金的基金经理信息
- 基金分析持仓信息
- 基金关系板块
- 标注基金市场内场外

| 角色       | 类型     | 权限描述                      |
| ---------- | -------- | ----------------------------- |
| 电力交易用户 | 人类用户 | 使用已注册的手机号/邮箱和密码登录电力交易平台 |
| 认证系统   | 系统     | 验证用户身份，生成JWT令牌，记录登录日志 |

## 🔁 流程步骤表格

| 步骤 | 步骤名称       | 执行方      | 输入数据                 | 输出数据                  | 失败处理说明              |
| ---- | -------------- | ----------- | ------------------------ | ------------------------- | ------------------------- |
| 1    | 访问登录页面   | 电力交易用户 | 浏览器访问登录URL        | 显示登录表单页面          | 页面加载失败时显示错误提示 |
| 2    | 输入登录信息   | 电力交易用户 | 账号（手机号/邮箱）、密码 | 表单数据提交              | 格式验证失败时显示提示信息 |
| 3    | 提交登录请求   | 前端系统    | 用户凭据、设备信息       | HTTP POST请求             | 网络异常时重试机制        |
| 4    | 验证用户身份   | 认证系统    | 用户凭据、密码哈希       | 身份验证结果              | 账号密码错误时显示错误信息 |
| 5    | 检查账号状态   | 认证系统    | 用户ID、账号状态         | 账号可用性确认            | 账号被禁用时显示禁用提示   |
| 6    | 生成访问令牌   | 认证系统    | 用户身份、权限信息       | JWT令牌、刷新令牌         | 令牌生成失败时返回系统错误 |
| 7    | 记录登录日志   | 认证系统    | 用户ID、IP地址、时间戳   | 登录记录保存              | 日志记录失败不影响登录流程 |
| 8    | 返回登录结果   | 认证系统    | Token、用户基本信息      | 登录成功响应              | 响应超时时前端提示重试    |
| 9    | 跳转交易平台   | 前端系统    | 访问令牌、用户权限       | 电力交易平台主页          | 权限不足时显示权限提示     |
| 10   | 加载用户数据   | 业务系统    | 用户ID、权限范围         | 个人信息、交易数据        | 数据加载失败时显示重试按钮 |

## 🛡️ 安全与合规

### 数据安全
- 🔒 用户密码使用bcrypt加密存储，禁止明文存储
- 🔒 敏感数据传输使用HTTPS加密
- 🔒 JWT令牌使用标准格式，设置合理过期时间
- 🔒 登录日志记录IP地址、设备信息、时间戳

### 访问控制
- 🔐 登录失败5次后锁定账户15分钟
- 🔐 同一IP地址1分钟内最多尝试10次登录
- 🔐 JWT令牌有效期设置为2小时，支持刷新机制
- 🔐 异常登录行为检测和告警

### 合规要求
- 📋 符合《网络安全法》用户身份认证要求
- 📋 符合《个人信息保护法》数据处理要求
- 📋 登录行为记录符合审计要求，保存180天
- 📋 用户隐私信息保护措施

### 日志审计
- 📊 记录所有登录尝试（成功/失败）
- 📊 记录用户IP地址、设备信息、时间戳
- 📊 记录异常登录行为（异地登录、设备变更）
- 📊 定期生成登录统计报告和异常分析

## 🔧 Agent 拆解任务

### 后端开发任务
- [ ] **任务1**: 创建电力交易用户认证相关数据表
  - 基于trade_operation_db设计实现用户信息表
  - 创建用户登录日志表（user_login_logs）
  - 创建用户会话表（user_sessions）
  - 创建JWT令牌黑名单表（token_blacklist）

- [ ] **任务2**: 实现用户认证API接口
  - POST /api/user/auth/login（电力交易用户登录）
  - POST /api/user/auth/logout（用户登出）
  - POST /api/user/auth/refresh（刷新JWT令牌）
  - GET /api/user/auth/profile（获取用户信息）
  - POST /api/user/auth/verify-token（验证令牌有效性）

- [ ] **任务3**: 实现密码安全机制
  - 密码bcrypt加密存储
  - 密码强度验证

  - 账号锁定和解锁机制

- [ ] **任务4**: 实现JWT令牌管理
