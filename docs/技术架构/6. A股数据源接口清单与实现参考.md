# A股数据源接口清单与实现参考

## 0. 定位

本文把根目录 `SKILL.md` 中的 A 股全栈数据能力整理为 TradeHub 可落地的接口清单和 Go 实现参考。它不是要求把 Skill 里的 Python 代码整体搬进项目，而是把其中验证过的数据源、端点、字段和风控经验纳入 TradeHub 的股票侧架构。

落地原则：

- 主体服务继续使用 Go：`tradehub-market-api` 负责第三方行情/资讯/信号源直连，`tradehub-stock` 负责统一 API、任务编排、入库和业务计算。
- Python 只保留为可选适配器：`mootdx` 这类 TCP 协议能力可以先通过已有 Python adapter 过渡，稳定后再评估 Go 原生实现或独立 sidecar。
- 可实时查询的数据优先不落库：快照、盘口、短周期 K 线可走缓存透传。
- 可分析、排行、回测、筛选的数据必须落库：龙虎榜、资金流、研报、公告、解禁、两融、涨停池、行业板块等统一走采集任务。
- 东财接口必须限流：串行、复用连接、带 UA/Referer，批量任务不允许并发扫东财。

## 5. Go 实现参考

### 5.1 market-api 适配器约定

新增数据源优先放在 `tradehub-market-api/cmd/market-api/internal/{source}`，每个 source 包保持以下约定：

```go
type Client struct {
	http *http.Client
}

func New(timeout time.Duration) *Client
```

请求实现要求：

- `http.Transport` 必须设置 `ProxyFromEnvironment`、连接池、TLS 超时和响应头超时。
- 所有请求必须使用 `context.Context`。
- 上游异常统一返回 error，不在 adapter 内吞掉。
- 解析后的结构体必须包含 `dataset`、`source`、`symbol/code`、`captured_at`。
- 原始字段复杂或仍在校准阶段时可保留 `raw`，但业务 API 不直接依赖 raw。

### 5.2 东财限流客户端

现有 `eastmoney.Client` 已做 host 轮换，但 datacenter、reportapi、search、push2ex 还需要统一限流。建议新增一个通用节流器：

```go
type RateLimitedClient struct {
	http       *http.Client
	minGap     time.Duration
	lastCallMu sync.Mutex
	lastCall   time.Time
}

func (c *RateLimitedClient) Do(req *http.Request) (*http.Response, error) {
	c.lastCallMu.Lock()
	wait := c.minGap - time.Since(c.lastCall)
	if wait > 0 {
		timer := time.NewTimer(wait)
		c.lastCallMu.Unlock()
		select {
		case <-req.Context().Done():
			timer.Stop()
			return nil, req.Context().Err()
		case <-timer.C:
		}
		c.lastCallMu.Lock()
	}
	c.lastCall = time.Now()
	c.lastCallMu.Unlock()
	return c.http.Do(req)
}
```

建议默认参数：

- 实时接口：`minGap=300ms`，只用于单票/低频。
- 东财 datacenter/reportapi 批量任务：`minGap=1200ms`。
- 大批量补数：通过任务队列串行执行，不开 worker 并发。

```go
	PageSize    int
}

func (c *Client) QueryDataCenter(ctx context.Context, q DataCenterQuery) ([]map[string]any, error)
```

落地位置：

- 上游调用：`tradehub-market-api/cmd/market-api/internal/eastmoney/datacenter.go`
- 任务编排：`tradehub-stock/internal/application/datacenter`
- HTTP 查询：沿用 `/api/stock/v1/datacenter/query/*`，新增两融、解禁、股东户数、分红查询。

### 5.4 统一任务模型

所有需要入库的数据都按任务执行：

| task_type | 数据范围 | 建议频率 |
| --- | --- | --- |
| `stock_lhb_daily` | 全市场龙虎榜 | 交易日收盘后 |
| `stock_money_flow_daily` | 全市场或自选池 120 日资金流 | 每日收盘后 |
| `stock_industry_rank_daily` | 行业板块排名 | 每 5-15 分钟或收盘快照 |
| `stock_research_report_sync` | 个股/行业研报 | 每日 1-2 次 |
| `stock_announcements_sync` | 巨潮公告 | 每日 1-2 次 |
| `stock_limit_up_pool_sync` | 涨停/炸板/跌停池 | 交易时段 1-5 分钟 |
| `stock_fundamental_sync` | 基本面、财报、分红、股东户数 | 每日或每周 |
| `stock_hot_rank_sync` | 人气榜/热榜 | 交易时段 5-15 分钟 |

任务执行必须记录：

- `source`：上游数据源。
- `source_endpoint`：端点或协议名。
- `source_params_hash`：请求参数 hash，便于去重。
- `trade_date` / `publish_date` / `snapshot_at`：业务日期。
- `raw_payload_path`：必要时保留原始响应落盘或对象存储路径。
- `rows_inserted`、`rows_updated`、`rows_skipped`、`error_message`。

## 8. 实施顺序

第一批做“高收益、低授权、适合 Go 直连”的能力：

1. 包装已有腾讯 ticks/order-book 到 `/api/stock/v1/market/*`。
2. 扩展东财 datacenter 通用查询，补齐两融、解禁、股东户数、分红。
3. 扩展东财 push2/push2his，补行业排名、概念归属、分钟/日级资金流。
4. 新增涨停池/炸板池/跌停池/昨涨停池，并在 Go 中计算炸板率、连板高度、晋级率。
5. 新增东财 reportapi 研报列表和个股新闻，先只存 metadata，不默认下载 PDF。
6. 新增巨潮公告检索，公告 PDF 下载作为异步任务。
7. 最后接入同花顺热点、北向、涨停原因、iwencai 等存在反爬或授权边界的能力。

## 9. 风险和边界

- 上游公开接口没有 SLA，必须保留 `source_health` 和降级响应。
- 东财批量补数必须串行限流，不能为了速度开并发。
- iwencai 需要 Key，不进入默认部署必需项。
- 财联社旧快讯不接入，避免把失效接口写进产品能力。
- 股票计算逻辑必须在 Go 的 domain/application 层实现；Python 不承担核心计算。
- 对研报、公告 PDF 等文件型数据，数据库只保存 metadata、hash、文件路径和解析状态。
