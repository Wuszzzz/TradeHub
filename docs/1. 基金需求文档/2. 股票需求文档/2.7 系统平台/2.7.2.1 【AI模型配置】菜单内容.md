## 📌 业务目标

AI模型配置菜单用于集中管理股票系统中的 AI 服务商、模型、API Key、Base URL、思考模式、联网工具、Token 限额和默认模型。该菜单支撑 AI 分析报告、AI 智能体、浮动 AI 助手、AI 推荐预警和市场资讯总结。

需要支持：

- DeepSeek
- 硅基流动
- 智谱AI
- 字节豆包
- 阿里云百炼
- Moonshot
- 腾讯混元
- 讯飞星火
- 零一万物
- MiniMax
- 百川智能
- 百度千帆
- OpenAI
- Azure OpenAI
- OpenRouter
- Ollama
- OpenAI 兼容接口
- Base URL
- API Key
- 模型名称
- 默认模型
- 启用状态
- 联通测试
- Token 限额
- 成本统计
- 故障切换

## 👤 主要参与角色

| 角色 | 类型 | 权限描述 |
| --- | --- | --- |
| 管理员 | 人类用户 | 配置模型服务和密钥 |
| 股票用户 | 人类用户 | 选择授权模型进行 AI 分析 |
| AI系统 | 系统 | 调用模型生成结果 |
| 审计系统 | 系统 | 记录模型调用和成本 |

---

## 🧩 业务流程图（文字描述）

1. 管理员进入【AI模型配置】菜单
2. 管理员新增模型服务商、Base URL、API Key 和模型名称
3. 系统加密保存密钥
4. 管理员点击联通测试
5. 系统调用模型测试接口并返回延迟、可用性和错误信息
6. 管理员设置默认模型、启用状态和 Token 限额
7. AI 分析功能读取可用模型列表
8. 用户选择模型发起 AI 分析
9. 系统记录模型调用、Token 成本和失败原因

## 🔁 流程步骤表格

| 步骤 | 步骤名称 | 执行方 | 输入数据 | 输出数据 | 失败处理说明 |
| --- | --- | --- | --- | --- | --- |
| 1 | 新增配置 | 管理员 | 服务商、模型、密钥 | 配置草稿 | 必填缺失提示 |
| 2 | 加密保存 | 后端系统 | API Key | 加密密钥 | 保存失败回滚 |
| 3 | 联通测试 | AI系统 | 配置参数 | 测试结果 | 失败显示错误 |
| 4 | 启用模型 | 管理员 | 配置ID | 启用状态 | 冲突提示 |
| 5 | 选择模型 | 用户 | 模型配置 | AI 请求 | 无权限提示 |
| 6 | 记录成本 | 审计系统 | Token 用量 | 成本记录 | 失败告警 |

## 📊 页面字段说明

| 字段 | 中文名 | 展示要求 | 筛选/排序 |
| --- | --- | --- | --- |
| provider | 服务商 | DeepSeek/OpenAI/Ollama 等 | 支持筛选 |
| model_name | 模型名称 | 可编辑 | 支持搜索 |
| base_url | 接口地址 | 脱敏展示 | 支持查看 |
| api_key | API Key | 加密脱敏 | 禁止明文展示 |
| enabled | 是否启用 | 开关 | 支持筛选 |
| is_default | 是否默认 | 是/否 | 支持筛选 |
| test_status | 测试状态 | 成功/失败/未测试 | 支持筛选 |
| latency_ms | 延迟 | 毫秒 | 支持排序 |
| token_limit | Token 限额 | 数值 | 支持配置 |
| monthly_cost | 月度成本 | 金额或点数 | 支持排序 |
| updated_at | 更新时间 | 精确到秒 | 支持排序 |
